## What to build

Set up the admin login flow, protected admin routes, and the main dashboard layout shell (sidebar, header). This provides the foundational UI where all other admin pages will live. 

## Acceptance criteria

- [ ] Admin authentication flow separate from customer auth (e.g., using NextAuth/Clerk or custom JWT)
- [ ] Protected route wrapper for `/admin/*`
- [ ] Admin dashboard shell layout with a sidebar navigation
- [ ] Placeholder empty pages for Orders, Products, and Delivery Fees

## Blocked by

- None - can start immediately
